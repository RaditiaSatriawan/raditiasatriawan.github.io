Panduan Definitif: Penyiapan Kluster AI Modern dengan Ray.io

Dokumen ini merangkum proses penyiapan dan troubleshooting kluster fisik Ray dari nol.

Arsitektur: 1 PC Master (Ubuntu) sebagai Head Node, 4 PC Slave (Ubuntu) sebagai Worker Node, terhubung di jaringan LAN pribadi. Beberapa node memiliki GPU.

Fase 1: Persiapan Fondasi & Lingkungan

Fase ini mengasumsikan Anda sudah menyelesaikan penyiapan Jaringan & OS dari panduan Spark/Hadoop. Fondasinya sama.

1.1. Atasi "Neraka Dependensi" dengan Conda

Untuk menghindari konflik versi Python dan pustaka (terutama untuk GPU), kita menggunakan lingkungan terisolasi dengan Conda. Ini adalah praktik terbaik.

DI SEMUA PC (MASTER & SEMUA SLAVE)

  1. Instal Miniconda: Unduh dan jalankan installer Miniconda3-latest-Linux-x86_64.sh. Saat ditanya `Do you wish the installer to initialize Miniconda3?`, jawab `yes`.
  2. Buat Lingkungan: Setelah membuka terminal baru, buat lingkungan khusus untuk ML. 
    conda create --name pytorch_legacy python=3.8 -y

Troubleshooting Conda

Error: CondaToSNonInteractiveError saat conda create.
Solusi: Jalankan perintah tanpa flag -y (conda create --name pytorch_legacy python=3.8) dan jawab yes secara manual saat diminta persetujuan Terms of Service.

Fase 2: Instalasi Ray & Pustaka Pendukung

Semua instalasi `pip` harus dilakukan di dalam lingkungan `conda` yang sudah diaktifkan.

DI SEMUA PC (MASTER & SEMUA SLAVE)

# 1. Aktifkan lingkungan
conda activate pytorch_legacy

# 2. Instal Ray dengan versi spesifik untuk konsistensi
pip install "ray[default]==2.10.0"

# 3. Instal pustaka ML lainnya
pip install -U "numpy<2" # Penting untuk kompatibilitas
pip install torch==1.7.1+cu92 torchvision==0.8.2+cu92 -f https://download.pytorch.org/whl/torch_stable.html # Untuk GPU
pip install lightgbm scikit-learn pandas pyarrow

Troubleshooting Instalasi Python

Error: ray: command not found
Solusi: Perintah Ray terinstal di ~/.local/bin yang tidak ada di `PATH` sesi non-interaktif. Solusi terbaik adalah menambahkan export PATH="$HOME/.local/bin:$PATH" ke file ~/.profile di semua mesin.

Error: numpy.dtype size changed atau numpy.core.multiarray failed to import
Solusi: Terjadi konflik antara NumPy versi 2.x yang baru dengan pustaka sistem yang lama. Solusinya adalah memaksa instalasi NumPy versi 1.x dengan pip install -U "numpy<2".

Error: No matching distribution found for torch
Solusi: Versi Python Anda (misal: 3.10) terlalu baru untuk versi PyTorch lama yang dibutuhkan oleh driver GPU lama Anda. Solusinya adalah dengan membuat lingkungan `conda` dengan Python versi lama (misal: 3.8) seperti yang dijelaskan di atas.

Fase 3: Menghidupkan Kluster Ray

Memulai kluster Ray jauh lebih sederhana daripada Spark.

3.1. Mulai Head Node

DI PC MASTER

Aktifkan `conda` lalu jalankan `ray start`. Kita gunakan --num-cpus=0 agar master hanya menjadi manajer.

conda activate pytorch_legacy
ray start --head --port=6379 --dashboard-host=0.0.0.0 --node-ip-address="10.42.0.1" --num-cpus=0

3.2. Hubungkan Worker Node

DI SETIAP PC SLAVE

Aktifkan `conda` lalu jalankan `ray start` dengan menunjuk ke alamat master.

conda activate pytorch_legacy
ray start --address='10.42.0.1:6379'

Troubleshooting Koneksi Ray

Error: Worker crash dengan error C++ Failed to get internal config: RpcError.
Solusi: Ini adalah masalah komunikasi. Pastikan: 1) Anda menggunakan IP LAN yang benar (`10.42.0.1`), bukan IP WiFi master. 2) Anda menggunakan parameter --node-ip-address di head node. 3) Versi Ray di master dan semua slave 100% identik (lakukan instalasi ulang dengan versi spesifik jika perlu).

Fase 4: Verifikasi & Pengujian

4.1. Verifikasi Kluster

Buka browser dan kunjungi **Ray Dashboard** di http://10.42.0.1:8265. Pastikan semua node terdaftar dan sumber daya (termasuk GPU) terdeteksi dengan benar.

4.2. Pengujian (Benchmark)

Jalankan skrip benchmark Anda (misalnya, `benchmark_brutal_ray.py`) dari master di dalam lingkungan `conda`.

conda activate pytorch_legacy
python3 benchmark_brutal_ray.py

Troubleshooting Benchmark Ray

Error: AttributeError: 'dict' object has no attribute 'dropna'.
Solusi: Kesalahan dalam kode. `map_batches` di Ray Data memberikan data sebagai kamus Python, bukan Pandas DataFrame. Perbaiki fungsi `preprocess` Anda dengan mengubah kamus menjadi DataFrame terlebih dahulu: df = pd.DataFrame.from_dict(batch).
